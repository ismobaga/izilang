**Evidence**: 
- Reviewed lexer.cpp implementation - proper skipLineComment() and skipBlockComment() methods

**Action Taken**: Updated STATUS.md to reflect accurate state

### 2. Wildcard Imports Already Implemented ✅
**Finding**: Wildcard imports (`import * as name from "module"`) create namespace objects (Maps).  
**Evidence**:
- Examined ImportStmt implementation in interpreter.cpp (line 393-395)
- Tested: `import * as math from "math"` successfully creates module object
- Can access via bracket notation: `math["sqrt"](16)` returns `4`
- All 68 tests passing including module system tests

**Action Taken**: Documented implementation status and usage in STATUS.md

### 3. VM is Functional, Not Broken ✅
**Finding**: VM passes all tests and is functional, just not recommended as default.  
**Evidence**:
- VM tests passing: `./bin/Debug/tests/tests "[vm]"` shows 5 assertions in 3 test cases passing
- Full VM implementation with 30+ opcodes
- Can be enabled via `--vm` flag

**Decision**: Marked VM as "experimental" rather than "broken", keep interpreter as stable default

## Design Decisions Documented

All critical decisions have been frozen and documented in `docs/DECISIONS.md`:

### 1. Execution Model (Section 2.1)
**Decision**: Hybrid execution model  
- **Default**: Tree-walking interpreter (production-ready)
- **Experimental**: Bytecode VM (opt-in via `--vm` flag)
- **Rationale**: Interpreter provides excellent debugging, VM provides future optimization path

### 2. Memory Management (Section 4.1)
**Decision**: Reference counting for v0.1  
- **Implementation**: std::shared_ptr
- **Rationale**: Already working, simple, deterministic, sufficient for v0.1
- **Limitations**: Cannot handle circular references (users must avoid)
- **Future**: Mark-and-Sweep in v0.3+

### 3. Backward Compatibility (Section 11.1)
**Decision**: SemVer with breaking changes allowed pre-v1.0  
- **Policy**: Breaking changes permitted in v0.x minor versions
- **Documentation**: All changes documented in CHANGELOG.md
- **Future**: Full SemVer compliance from v1.0+

### 4. Standard Library Namespacing (Section 8.2)
**Decision**: Accept current behavior for v0.1  
- **Named imports**: Add to global scope
- **Wildcard imports**: Create namespace objects (Map)
- **Access**: Bracket notation `module["fn"](args)`
- **Future**: Add dot notation in v0.2 (`module.fn()`)
